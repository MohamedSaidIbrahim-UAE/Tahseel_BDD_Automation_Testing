# Professional Test Scenarios – Financial Reports  
**Module:** Reports → Financial Reports (تقارير مالية)  
**Version:** Staging (2026-06-08)  
**Objective:** Validate all 14 financial reports with focus on RBAC, filters, calculations, integration with external systems, data accuracy, export/print, and error handling.

---

## 1. Role-Based Access Control (RBAC) Scenarios  
*These scenarios apply to all financial reports unless otherwise noted. They ensure only authorised roles can access and view the reports.*

| Scenario ID | Type | Focus Area | Scenario Description | Expected Result |
|-------------|------|------------|----------------------|-----------------|
| FIN-RBAC-01 | Positive | RBAC | Login with a role that has explicit permission to the financial reports group (e.g., Finance Admin). Navigate to each of the 14 reports. | All reports appear in the menu and load successfully with data. |
| FIN-RBAC-02 | Negative | RBAC | Login as a restricted role (e.g., Money Loader User, Seller). Attempt to access any financial report via direct URL (e.g., `/report-show/...`). | User is redirected to the dashboard or shown an “Access Denied” error; no report data is rendered. |
| FIN-RBAC-03 | Negative | RBAC | Login as an entity-specific user (e.g., a government department user with scope limited to own entity). Open a report that can be filtered by revenue entity, e.g., “Transaction Fees For All Payment Methods”. Verify entity dropdown. | The dropdown either shows only the user’s own entity or automatically applies that filter; no data from other entities is displayed. |
| FIN-RBAC-04 | Negative | RBAC | A user without “Export” privilege attempts to export a settlement report (PDF/Excel). | The export button is hidden or returns an authorisation error. |
