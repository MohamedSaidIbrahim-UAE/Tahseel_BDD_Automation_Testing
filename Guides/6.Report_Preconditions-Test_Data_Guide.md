# Report Preconditions & Test Data Guide  
**Tahseel Manage Portal – Reports Module**  
*Prioritised for efficient test preparation (P1 → P2 → P3)*  

This guide outlines the necessary preconditions, master data, and test data sets required to effectively test each report. It is organised by report priority and highlights common dependencies that can be shared across multiple reports to minimise setup effort.

---

## 1. Common Preconditions & Shared Test Data  
*Applicable to virtually all reports. Set these up once before any report testing begins.*

| Category | Precondition / Test Data | Why It Is Needed |
|----------|--------------------------|------------------|
| **Users & Roles** | Create test users for each role: Finance Admin, Entity Accountant (A/B), Collector, Money Loader, Merchant, Tasheel Center Manager, Customer Support, etc. | RBAC validation across all reports. |
| **Base Entities** | Define at least two revenue entities (e.g., Entity‑A, Entity‑B) and two deposit entities. | Used in almost every report (filtering, entity-scoped access). |
| **Services & Fees** | Create a set of services: taxable service SRV‑TAX (fee 100 AED), exempt service SRV‑EXM (0 AED), service with dependant fees (e.g., Express +30 AED), book sale service, safari tickets, etc. | All transaction-based reports require services. |
| **Payment Methods** | Enable all payment methods: Credit Card (MCC‑1234, merchant fee 2.5%), Direct Debit (1% fee), Cheque, E‑Voucher, Cash. | Reports grouped by payment method need multiple methods. |
| **Banks & External Systems** | Configure test bank gateways (Mashreq, SIB, Central Bank) in staging. Provide a set of valid bank files and a set of corrupt/mismatched files. | Mandatory for all settlement & bank transfer reports. |
| **Date & Time** | Ensure the staging environment’s system clock is fixed at a known date (e.g., 2026‑06‑10) and all test data uses that date or a controlled range. | Predictable date filters and settlement periods. |

## 5. Quick Start: Data Preparation Sequence  
1. **Base Data** (Day 1): Create entities, users, services, payment methods, banks.  
2. **Instrument Data** (Day 1-2): Issue cards, create accounts, load initial balances, generate e‑voucher batches.  
3. **Transaction Data** (Day 2-3): Post a rich set of transactions covering all payment methods, services, entities, and statuses. Include some cancellations, refunds, and deposits.  
4. **Bank Files** (Day 3): Prepare matching and mismatched settlement files for all external banks.  
5. **Workflow & Status Changes** (Day 3-4): Process approvals, refunds, retentions, card blocks, permit suspensions – to populate logs and lifecycle reports.

With this guide, the test environment can be efficiently configured to validate every report, respecting priority and minimising redundant setup.
